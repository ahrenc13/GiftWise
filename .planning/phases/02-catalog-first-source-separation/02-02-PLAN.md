<objective>
Scope eBay searches to only the interests where the DB has thin coverage. After this plan:
- eBay fires only for interests where `per_interest_counts[interest] < 5`
- Interests absent from `per_interest_counts` are treated as 0 (always fire eBay for them)
- Total eBay contribution is capped at `EBAY_NICHE_CAP = 7` items
- If all interests have >= 5 DB products, eBay is skipped entirely
- `per_interest_counts = {}` pre-initialized before the DB try block (prevents NameError on DB failure)

Purpose: eBay currently fires for all profile interests. With a healthy DB (11,000+ products from nightly sync), most interests already have good coverage. Firing eBay only for gaps reduces session latency and prevents eBay from dominating the inventory pool with marketplace listings that lack affiliate tracking.

Output: `multi_retailer_searcher.py` only — 3 targeted changes.
</objective>
