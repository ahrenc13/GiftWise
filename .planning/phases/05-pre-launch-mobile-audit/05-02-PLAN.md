Read templates/recommendations.html fully before making any changes. Then apply two additions:

**ADDITION 1 — HTML: sticky bar div**

Find the closing `</div>` tag that ends the main `.container` div (around line 969, after the `{% endif %}` that closes the share section). Immediately AFTER that closing container div and BEFORE the `<script>` tags, insert:

```html
<!-- Floating share bar: mobile only, always visible -->
<div class="mobile-share-bar" id="mobile-share-bar">
    <button class="mobile-share-btn" onclick="shareViaText(false)">
        📱 Text this list
    </button>
</div>
```

This div is rendered unconditionally (shown to both locked and unlocked users). The shareViaText(false) call matches the unlocked state button — it shares the current page URL directly. It is acceptable to call this for locked users too since navigator.share() shares the URL regardless of lock state; the experience is the same.

**ADDITION 2 — CSS: styles for the sticky bar**

Inside the `<style>` block in recommendations.html, add the following CSS. Place it AFTER all existing desktop rules and BEFORE (or appended to) the existing `@media (max-width: 768px)` block:

```css
/* Mobile sticky share bar — desktop hidden */
.mobile-share-bar {
    display: none;
}

@media (max-width: 768px) {
    .mobile-share-bar {
        display: flex;
        position: fixed;
        bottom: 0;
        left: 0;
        right: 0;
        z-index: 50;
        background: rgba(255, 255, 255, 0.95);
        backdrop-filter: blur(8px);
        -webkit-backdrop-filter: blur(8px);
        padding: 12px 20px;
        padding-bottom: calc(12px + env(safe-area-inset-bottom));
        border-top: 1px solid #e5e7eb;
        justify-content: center;
    }

    .mobile-share-btn {
        background: linear-gradient(135deg, #10b981, #059669);
        color: white;
        border: none;
        border-radius: 10px;
        padding: 14px 32px;
        font-size: 16px;
        font-weight: 700;
        cursor: pointer;
        width: 100%;
        max-width: 360px;
        min-height: 44px;
    }
}
```

Key decisions in this CSS:
- `display: none` at desktop level: sticky bar is invisible on anything wider than 768px
- `z-index: 50`: below the card modal (z-index: 100+) and card-backdrop — card expansion still works correctly
- `bottom: 0`: sits flush at the bottom, not at bottom: 20px (avoids overlapping the OneSignal banner at bottom: 20px if both are showing)
- `padding-bottom: calc(12px + env(safe-area-inset-bottom))`: handles iPhone home indicator safe area automatically (env() is ignored on Android and older iOS where it equals 0)
- `backdrop-filter: blur(8px)`: frosted glass effect so recommendation text behind the bar doesn't bleed through; -webkit prefix for Safari
- `max-width: 360px` on button: keeps the button from stretching absurdly wide on landscape phone or tablet at exactly 768px

Do NOT modify any existing CSS rules. Do NOT touch the existing share section at lines 912-968. Do NOT change shareViaText() or any other JS.
