<objective>
Fix the five undersized touch targets and the cramped actions-section padding in `recommendations.html`. These are CSS-only changes — no JS, no routing, no template logic.

Purpose: TikTok visitors will be on mobile. Tap targets below 44px cause mis-taps and frustration. Apple HIG and Android Material both require 44px minimum. Fixing this before traffic arrives is low-effort, high-polish.

Output: Updated `templates/recommendations.html` with `min-height: 44px` (and display corrections) on five button selectors plus a mobile `@media` override for `.actions-section` padding.
</objective>

<task type="auto">
  <name>Task 2: Add mobile padding override for actions-section</name>
  <files>templates/recommendations.html</files>
  <action>
The `.actions-section` currently has `padding: 50px 40px` with no mobile override. On a 375px phone this leaves only 295px of usable content width — visible but cramped.

Find the existing `@media (max-width: 768px)` block (lines ~608–626). Add the following rule inside it (after the existing `.header` rule):

```css
.actions-section {
    padding: 32px 20px;
}
```

Why 32px vertical / 20px horizontal:
- 32px vertical: matches the container's own 20px horizontal padding rhythm while giving enough breathing room
- 20px horizontal: matches `.container`'s own horizontal padding (20px), so the section content aligns with the page edge — no extra squeeze

Do NOT add a second `@media` block if one already exists for `max-width: 768px` — append inside the existing one.
